Multi-site Analytics Layer (GTM → GA4 → Looker Studio)

Master GTM container across sites, one GA4 property with multiple streams, and production dashboards for Website Performance, SEO, and Attribution. Built for fast onboarding and trustworthy KPIs.

Overview Dashboards Docs

What I built

  • GTM master container with standardized events (form_submit, phone_click, email_click, engagement).
  • GA4 single property, multiple data streams (one per site), unified taxonomy and conversions.
  • Looker Studio dashboards: Website Performance (GA4), SEO Insights (GSC), Attribution & Growth.
  • New-site onboarding checklist – add stream, update GTM lookups, publish, validate in DebugView, dashboards auto-populate.

Tracking foundations

GTM Master Container

Reusable tags/triggers/variables; Lookup Tables for site-specific IDs; form tracking with form_name; click tracking for tel/mail/CTA; UTM normalization; optional hidden site_id field on forms for cross-site attribution.

GA4 Property & Streams

One property, multiple streams (Site A/B/C). Enhanced Measurement + custom dimensions: site_id, form_name, page_type. Conversions mapped to generate_lead, form_submit, and key CTA clicks.

Onboarding a new site (6 steps)

  1. Create GA4 stream → grab Measurement ID.
  2. Update GTM lookups/variables and deploy.
  3. Install GTM snippet on the new site.
  4. Test in GTM Preview + GA4 DebugView.
  5. Add site_id hidden field to forms if used.
  6. Confirm dashboards populate (filters by site_id).

Dashboards
