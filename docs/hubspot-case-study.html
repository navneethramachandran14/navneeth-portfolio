CRM & Automations (HubSpot + Make)

Architected and implemented a production CRM & marketing automation ecosystem with HubSpot, Make.com, and a unified analytics layer (GTM, GA4, Looker Studio) for a multi-location services business in Sweden.

Context & Problem

Leads arrived through several local websites and phone/email, but there was no central CRM. Quotations and bookings were handled manually in inboxes, ROT/Grön teknik admin was ad-hoc, and analytics were split by site. The goal was a single, scalable system covering capture → pipeline → automation → reporting, with the ability to onboard new local sites quickly.

My Role

  • Designed the end-to-end architecture (HubSpot ↔ Make ↔ Slack/Calendar/Sheets ↔ GTM/GA4 ↔ Looker Studio).
  • Built the HubSpot data model and pipelines: Lead → Quote → Booking → ROT/Grön teknik → Invoicing → Closed.
  • Created forms/templates (Quote Approval, ROT, Egenkontroll, Call Request) with hidden site_id for multi-site attribution.
  • Developed Make scenarios for new leads (Privat/Batteri/Laddbox and Företag/BRF), Quote→Booking with calendar invites, ROT & Grön teknik, Egenkontroll follow-ups, Slack alerts, and reminders.
  • Set up a master GTM container and a single GA4 property with multiple data streams; Looker dashboards for Website, SEO, and Attribution.
  • Wrote handover docs and operational standards (e.g., booking field formatting, new-site onboarding checklist).

Solution Overview

HubSpot CRM

Custom objects/properties, standardized lifecycle and pipeline stages, and form templates with site_id to tag the source site automatically. Sales can move deals from Quote to Booking with a single action.

Make.com Automations

Scenarios for new lead intake, quote approval → booking (technician invite + Slack alerts), ROT/Grön teknik administration, Egenkontroll emails, and periodic nudges. Google Sheets/Data Store for light data persistence where needed.

Analytics (GTM + GA4 + Looker)

One master GTM container across sites with consistent events (form_submit, phone_click, email_click, engagement). GA4 uses one property with multiple streams; site_id enables site-level filters in dashboards.

Operational Standards

Booking field formatting for clean calendar invites; 6-step new-site onboarding: create GA4 stream → update GTM lookups → publish → install GTM → test in DebugView → confirm dashboards.
