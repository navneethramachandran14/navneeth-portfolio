End-to-End HubSpot CRM & Automations

Architected and implemented a production CRM & marketing automation ecosystem with HubSpot, Make.com, and a unified analytics layer (GTM, GA4, Looker Studio) for a multi-location services business in Sweden.

Context & Problem

There was no central CRM, lead handling was manual (forms, emails, phone calls), and analytics were siloed by site. The business needed a scalable foundation that unified capture → CRM → automation → reporting across present and future local websites.

My Role

  • Designed the end-to-end architecture (HubSpot ↔ Make ↔ Slack/Calendar/Sheets ↔ GTM/GA4 ↔ Looker Studio).
  • Implemented HubSpot objects, forms, property model, templates (incl. hidden site_id).
  • Built Make scenarios for new leads, quote approval → booking, ROT/Grön teknik, self-inspection, reminders and alerts.
  • Set up a master GTM container, GA4 with one property & multiple streams, and Looker Studio dashboards.
  • Produced handover docs and operational guidelines (e.g., calendar/booking field formatting).

Solution Overview

HubSpot CRM

Custom pipeline stages (Lead → Quote → Booking → ROT/Grön teknik → Invoicing → Closed). Forms include Quote Approval, ROT, Egenkontroll, and Call Request. Hidden fields carry site_id for multi-site attribution.

Make.com Automations

Scenarios for New Lead (Privat/Batteri/Laddbox & Företag/BRF), Slack alerts, Quote → Booking with technician calendar invites, ROT & Grön teknik flows, Egenkontroll follow-ups and thank-you emails. Google Sheets/Data Store used where appropriate.

Analytics (GTM + GA4 + Looker Studio)

Single GTM master container across sites with event coverage: form_submit, phone_click, email_click, and engagement. GA4 uses one property with multiple streams; site_id enables site-level filtering. Looker Studio dashboards for Website Performance, SEO, and Attribution.

Operational Standards

Guidelines for consistent calendar invite formatting, and a documented 6-step flow to onboard new sites (create GA4 stream, update GTM lookups, publish, install GTM, test in DebugView, confirm dashboards).
